**项目名称**： vocal- calendar（语音日历工具）

**目标**：从零开发一个语音版的日历工具，运行在web端，遵循Superpowers全流程规范，确保可维护性和最佳实践。

**核心流程**：
brainstorming(澄清需求) → writing-plans → subagent-driven-development → test-driven-development → requesting-code-review → 交付

**关键节点确认**：

1. 设计文档确认（brainstorming后）
2. 开发计划确认（writing-plans后）
3. 测试覆盖率确认（test-driven-development后）

**功能需求**：

一、语音交互核心
  ├── 语音添加 / 删除 / 修改 / 查询事件
  ├── 自然语言时间解析（"下周三"、"后天"等）
  ├── 多轮对话上下文 & 追问补全
  ├── 语音指令历史记录
  └── 语音指令帮助面板
二、日历管理
  ├── 日 / 周 / 月 / 议程 四视图
  ├── 事件 CRUD（标题、时间、分类、优先级、备注、地点）
  ├── 点击空白格快速创建事件
  ├── 拖拽调整事件时间（Web 独有）
  ├── 右键上下文菜单（Web 独有）
  ├── 重复事件（每天 / 每周 / 每月）
  └── 全文搜索事件
三、智能提醒
  ├── 多层级提前提醒
  ├── 浏览器 Push 通知（后台也能提醒）
  └── 语音播报提醒
四、AI 智能功能
  ├── 智能冲突检测 & 空闲时段推荐
  └── 每日语音播报摘要（早间 & 晚间）
五、UI/UX 交互
  ├── 语音录音波形可视化
  ├── 实时语音转文字
  ├── 键盘快捷键（Web 独有）
  ├── 深色 / 浅色主题
  └── PWA 支持（可安装到桌面）
六、增值功能
  ├── 天气集成
  ├── 节假日 / 调休标注
  ├── 时间花费统计图表
  ├── 分享只读日程链接
  └── 打印 / 导出 PDF
七、数据与账户
  ├── JWT 用户认证（注册 / 登录）
  ├── 数据云端存储
  └── iCal 格式导入 / 导出

**技术栈**：

前端
层级	技术	用途
框架	React 18 + Vite 6	核心框架 + 构建工具
路由	React Router v6	页面路由管理
状态管理	Zustand	全局状态（轻量，比 Redux 简洁）
样式	Vanilla CSS + CSS Variables	主题系统、动画、全局设计 token
日历组件	FullCalendar React	日/周/月/议程视图，支持拖拽
语音识别	Web Speech API	浏览器原生语音转文字（免费）
语音合成	Web Speech API TTS	浏览器原生语音播报

后端
层级	技术	用途
框架	Python FastAPI	异步高性能 API 服务
数据库	PostgreSQL 16	主数据存储
ORM	SQLAlchemy 2.0 + Alembic	数据库映射 + 迁移管理
缓存	Redis	会话管理 + 任务队列
定时任务	APScheduler	事件提醒定时触发
认证	JWT（PyJWT）	无状态用户认证
实时通信	FastAPI WebSocket	实时提醒推送
数据校验	Pydantic v2	请求/响应数据校验

AI / 语音处理
能力	技术	说明
语音识别	Web Speech API（主） / 科大讯飞（备）	中文口语转文字
自然语言理解	通义千问 Qwen API	口语 → 结构化日历指令解析
备选 LLM	OpenAI GPT-4o-mini / 智谱 GLM-4	可切换

部署
层级	技术	说明
容器化	Docker + Docker Compose	前后端 + 数据库统一编排
Web 服务器	Nginx	反向代理 + 静态资源托管
云服务器	阿里云 ECS（2核4G）	国内访问速度快，评审人员友好
HTTPS	Let's Encrypt（Certbot）	免费 SSL 证书，Web Speech API 必需
